目录结构：
aztec/
├── aztec2.0.sh              # 主脚本
├── .env                     # 交互生成的节点配置
├── docker-compose.yml       # 容器编排配置
├── bls_keys/                # CLI 注册时保存的密钥输出
├── logs/                    # 可选：手动收集日志
└── backups/                 # 可选：手动备份配置

运行容器产生的数据会放在 ~/.aztec/testnet/data/（archiver、world_state 等目录）


1 安装并启动节点：跟着提问一步步填信息，脚本写好配置并把容器拉起来。
2 看实时日志：立刻进入实时日志界面，随时 Ctrl+C 返回。
3 改日志级别：选想看的日志量级，脚本保存设置并提醒是否重启。
4 升级容器：停旧容器、拉新镜像、再启动，全程托管。
5 查节点状态：做一轮健康检查——容器、区块高度、P2P连接等，一眼看出好不好。
6 轻量删除：清容器和同步数据，但保留 .env 和 P2P 身份，下次可直接重装。
7 CLI 注册序列器：提示你准备好旧私钥、新密钥，帮你完成授权和注册流程。
8 重新加载 P2P 身份：确认你换好密钥文件后，脚本重启容器并检查新的节点 ID。
9 退出（或 q）：返回 shell，不做其他操作

针对当前的 2.1.2 版本：
如果你还没有在链上注册新的序列器身份，建议先跑一次选项 7：它会帮你生成/校验新密钥、完成 STAKE 授权并提交注册交易，让新的验证者身份准备好。等链上注册确认后，
再回到菜单用选项 1 做完整安装部署，这样脚本在生成 .env 和启动容器时，就能直接写入已经生效的密钥和地址，避免重复修改配置或重启
